# Contributing to Hunch

Thanks for your interest — contributions are very welcome.

Hunch is a **Claude Agent Skill**: a folder of Markdown instructions (`hunch/SKILL.md` plus `hunch/references/*.md`) that tells Claude how to build idea-validation landing pages, design systems, and ad creatives. There is no build step and no runtime server — improving Hunch means improving the instructions.

## Good first contributions
- **New design languages** — add to `hunch/references/design-languages.md` (each is a "style card": when-to-use + radius/border/shadow/button/surface/type/motion + a hero/video idiom).
- **Better playbooks** — sharpen `references/copy-craft.md`, `principles.md`, `discovery-seo-geo-aeo.md`, or `ui-ux-motion.md`.
- **Brand assets** — keep `references/brand-assets.md` accurate (official logos, sign-in/badge rules).
- **Docs** — clarify the README or the landing page in `docs/`.
- **Examples** — share a great validation page Hunch produced (sanitized).

## How to propose a change
1. Fork the repo and create a branch.
2. Make your edit. If you change `hunch/SKILL.md` or a reference, keep it concise and consistent with the existing voice.
3. Open a pull request describing **what** changed and **why** it makes Hunch's output better.

## Principles to keep
- **Honesty first.** Hunch never fabricates proof, testimonials, statistics, or scarcity. Don't add anything that does.
- **Don't look AI-generated.** Favor original, distinctive output over templated defaults.
- **English** for all skill content and deliverables.
